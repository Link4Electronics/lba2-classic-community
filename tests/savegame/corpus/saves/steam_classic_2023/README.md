# Steam classic save corpus (50 saves)

Reference regression corpus for the savegame load path. Drives the Layer-3
harness in `tests/savegame/corpus/run_harness.py`.

## Provenance

- **Source build:** Steam re-release of *Twinsen's Odyssey* (LBA2),
  executable `TLBA2C.exe` ("Classic" variant), Steam app `397330`.
- **Origin filesystem path on the contributor's machine:**
  `C:\Users\<user>\Saved Games\2point21\tlba2-classic\Save\`.
- **Capture window:** Sep 2022 – Apr 2025 (mtime range on the source
  files; some are mid-quest checkpoints, some are autosaves, one is
  `current.lba`).
- **Format:** version-byte `0xa4` (NumVersion 36 + `SAVE_COMPRESS`),
  LZSS-compressed payload, 32-bit retail wire ABI throughout. Each file
  is a complete checkpoint the engine wrote at runtime.
- **License:** Save files are runtime game state generated by the
  contributor playing a legitimately-purchased copy of the Steam
  re-release. They contain no retail game assets — only references
  (cube/scene IDs, body IDs, animation indices) into HQR archives that
  ship with the retail game. Distributed for the purpose of regression
  testing the engine's load path; not redistributable as game content.

## Anonymization

Two saves carried personal player-name strings in their headers
(everything else in the corpus uses LBA2 location/quest names like
`Ball of Sendell`, `Dark Monk`, `School of Magic` — game content, not PII).

The two PII-bearing saves were anonymized by patching the embedded
player-name field in place, preserving byte length so the rest of the
file (compressed-size header, LZSS payload, every offset) is bit-for-bit
unchanged. Verified post-patch: both still load with `flagload=0`.

| Original filename | Original embedded name | Renamed to | Patched embedded name |
|---|---|---|---|
| `Kerry.LBA` | `Kerry` (5 chars) | `Anon1.LBA` | `Anon1` (5 chars) |
| `Ivan.LBA` | `Ivan` (4 chars) | `Anon2.LBA` | `Anon` (4 chars) |

## Why this corpus matters

It exposed two struct-layout bugs the original 32-save corpus did not
catch:

1. **`T_EXTRA` 32-bit wire ABI.** `Dark Monk.LBA` (cube 93) crashes the
   parser at the `NbZones > MAX_ZONES` check after the `ListExtra` read
   over-runs the cursor by 12 bytes per extra (`U8 *PtrBody` is 4 bytes
   on 32-bit retail, 8 + 4-pad on 64-bit hosts).
2. **`S_PART_FLOW` 32-bit wire ABI.** `Off to be a Wizard.LBA` (cube 27)
   trips `wbyte2 > MAX_FLOW_DOTS` on flow #5 after the per-flow read
   over-runs by 4 bytes per flow (`S_ONE_DOT *PtrListDot` slot
   widening).

Both were fixed by `T_EXTRA_WIRE32` and `S_PART_FLOW_WIRE32` in
`SOURCES/SAVEGAME.CPP`. The harness is the regression contract: any
future change that breaks loading on this corpus is a regression on
real Steam-released retail saves.

## Expected harness outcomes

```
=== abi=auto ===
  ok_init              50

=== abi=32 ===
  ok_init              50
```

(`auto` = stride heuristic + retry; `32` = forced 32-bit wire ABI via
`LBA2_SAVE_LOAD_ABI=32`. Both should hit 50/50 on this corpus.)

## How to run

```bash
export LBA2_SAVE_TEST_DIR=$PWD/tests/savegame/corpus/saves/steam_classic_2023
LBA2_SAVE_DECOMPRESS=$PWD/build/tools/save_decompress \
    python3 scripts/save_probe.py --recursive --json-lines \
    "$LBA2_SAVE_TEST_DIR" > tests/savegame/corpus/probe.ndjson
python3 tests/savegame/corpus/build_manifest.py
python3 tests/savegame/corpus/run_harness.py --timeout 20 \
    --game-dir /path/to/LBA2 --lba2 $PWD/build/SOURCES/lba2
```

Retail data (`LBA2/` with `lba2.hqr`) is required — the engine has to
boot to the menu before replaying the load. The harness output is
written to `tests/savegame/corpus/manifest.json`.
